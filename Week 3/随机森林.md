## 随机森林分类
### 有什么重要参数？
- **树层面的参数:**

| 参数名                     | 机制逻辑                                                                                | 人话                        |
| ----------------------- | ----------------------------------------------------------------------------------- | ------------------------- |
| **`criterion`**         |                                                                                     |                           |
| **`max_depth`**         | 控制单棵决策树能生长的最大深度                                                                     | 其是最常用的剪枝参数，可以防止决策树太深而记住噪声 |
| **`max_features`**      | 当一棵决策树在某个节点上寻找最佳分裂点时，它不会考虑所有特征，而是**随机选取 `max_features` 个特征**作为候选，然后从这少量候选特征中找出最佳分裂点 |                           |
| **`min_samples_split`** | 一个节点必须包含**至少 `min_samples_split`** 个样本，才允许被继续分裂                                     |                           |
| **`min_samples_leaf`**  | 一个叶子节点必须包含**至少 `min_samples_leaf`** 个样本                                             |                           |
- **森林层面的参数：**

| 参数名                | 机制逻辑                                                      | 人话  |
| ------------------ | --------------------------------------------------------- | --- |
| **`n_estimators`** | 指定**森林中树的总数**-->在训练时，算法会独立地、依次构建 **`n_estimators`** 棵决策树。 |     |
| **`bootstrap`**    | 决定是否使用**有放回抽样** 来为每棵树构建训练集                                |     |

### 有什么重要属性？
- **`.estimators_`**：**查看森林列表**
    - **逻辑**：这是一个列表，里面装着 `n_estimators` 个 DecisionTree 对象，你可以单独把某一棵树拎出来看
- **`.oob_score_`**：**袋外得分**
    - **逻辑**：当参数 `oob_score=True` 时才有值 --> 它是模型在“没见过的数据”上的表现，约等于测试集得分
- **`.feature_importances_`**：**特征重要性**
    - **逻辑**：基于基尼不纯度减少量计算 --> 数值越大，说明该特征在分裂中贡献越大

### 有什么重要接口？
- **`fit(X, y)`**：**训练模型**
- **`predict(X)`**：**返回分类结果**
- **`apply(X)`**：返回样本所在的叶子节点索引
